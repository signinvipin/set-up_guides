Setup JupyterHub

First create a kubernetes cluster on azure

after deploying cluster,
> open bash CLI 

> az account list
Get a list of subscriptions for the logged in account. (autogenerated)

> az account show
Get the details of a subscription. (autogenerated)

> if multiple accounts then set a subscription to use for cluster
az account set --subscription mysubscription
Set a subscription to be the current active subscription. (autogenerated)

> Download cluster credentials
az aks get-credentials --resource-group test_group --name jupyterHubCluster --overwrite-existing

> We can check  present pods status
kubectl get pods -A
or, 
kubectl get po -A

Manual installation of jupyterhub on kubernetes: 

add jupyterhub to our repository
helm repo add jupyterhub https://jupyterhub.github.io/helm-chart/

update from repository
helm repo update

generate token 
openssl rand -hex 32

copy generated hex token to newly created config.yaml
touch config.yaml
vi config.yaml
as below

proxy:
  secretToken: <token>

Run below command,
RELEASE=jupyterhub
NAMESPACE=jhub
helm upgrade --cleanup-on-fail \
--install $RELEASE jupyterhub/jupyterhub \
--namespace $NAMESPACE \
--create-namespace \
--values config.yaml

check pods for available pods
kubectl get pods -A

Verify active service 
kubectl get services -A

go to 'services and ingresses' and click the external ip
to go to jupyterHub Server or Dashboard

### (for production)
To get https instead of http for jupyterhub site
we add below content as 
proxy:
  secretToken: "830561338d3df4c2166e2bec074bbb05b397d7bfd7c7ea5e87509bb0d883f3f7"
  https:
    enabled: true
    hosts:
      - your.domain.name
    letsencrypt:
      contactEmail: youremail@example.com

To get domain name go to sites like GoDaddy, Namecheap buy domain name.
Then login to GoDaddy, Namecheap sites and 
Go to DNS settings or DNS management for domain and Add a new record like below
  Type: A
  Name/Host: "@" for root domain, or "www" for the www subdomain
  Value/IP: Your external public IP address (e.g., 4.174.199.94)
  TTL: Leave default, or set as needed. 
Save the DNS record

and add it to config.yaml under hosts.
